## Enviroment
Use the following for logging and output
    from util.io_helpers import log, out
    log("logging message","filename.log")
    out("output message","filename.txt")

Write new classes in src/ and tests in script/
There is a src/__init__.py file, so don't worry about finding files there.


## Coding Style
Write the code in a clear, linear, step-by-step style that prioritizes readability and future adaptation over conciseness or cleverness. 
Prefer explicit helper functions, simple control flow, and modular structure so each step of the logic is easy to understand and modify later.


## Gang_of_Bandits class API
Bulk version of Bandit for running many bandits in parallel.

Use via
    from src.bandits import Gang_of_Bandits

Initialization
    Gang_of_Bandits(n_bandits, n_arms, distribution, means=None, gap=None)

Parameters
    n_bandits (required) → number of independent bandits simulated in parallel
    n_arms (required) → number of arms
    distribution (required) → "gaussian" or "bernoulli"
    means (optional) → matrix of fixed means (n_bandits x n_arms)
    gap (optional) → reward gap to enforce descending means
    Neither means nor gap → random means generated

Pulling arms
    bulk_pull(arm_index)

Parameters
    arm_index → either:
        int (same arm for all bandits)
        np.ndarray shape (n_bandits,) (one arm per bandit)

Returns
    rewards: np.ndarray shape (n_bandits,)


## ETCBulkAlgorithm class API
Bulk Explore-Then-Commit algorithm.

Use via
    from src.etc_bulk import ETCBulkAlgorithm

Initialization
    ETCBulkAlgorithm(bandit, exploration_rounds)

Parameters
    bandit (required): instance of Gang_of_Bandits
    exploration_rounds (required): int

Executing one step
    step()

Returns
    chosen_arms: np.ndarray shape (n_bandits,)
    rewards: np.ndarray shape (n_bandits,)

Behavior
    During exploration, arms are pulled round-robin.
    After exploration, each bandit commits to its best empirical arm.
